Análise por Player

🔴 Getnet (Santander)

✅

Prós

Pix 0% para correntistas Santander · Antifraude e 3DS inclusos · Data Center TIER IV · Taxas competitivas (3,49% crédito 1x) · Domicílio em qualquer banco

❌

Contras

API funcional mas menos polida que concorrentes · Homologação obrigatória com time Getnet (processo lento) · Documentação fragmentada · Não é dev-first

🔵 Mercado Pago

✅

Prós

D+0 instantâneo — dinheiro na hora · Pix 0% · Taxas agressivas no início (0,74% débito, 8,99% 12x) · API madura com SDK oficial · Onboarding em minutos · MCP Server para IA · Discord community

❌

Contras

Após período promocional, taxas sobem muito (12x salta para 22,59% no tier baixo) · Ecossistema fechado (dinheiro fica na conta MP) · Menos flexível para split avançado · Checkout Pro redireciona para MP

🟢 Pagar.me (Stone)

✅

Prós

API mais robusta e flexível do Brasil · docs.pagar.me excelentes · Retentativa transparente (salva 5-10% de vendas recusadas) · Split nativo (marketplace-ready) · Pix completo · Checkout hosted ou transparente · SDKs em 5 linguagens · LLMs.txt para IA

❌

Contras

Taxas de balcão altas (~25% em 12x) — precisam ser negociadas para volumes > R$50k · Antifraude pago à parte · Suporte às vezes lento · Requer conta transacional para Pix

E o Stripe?

🚫 Stripe — Não recomendado agora Embora a API do Stripe seja a melhor do mundo, no Brasil ela tem limitações críticas:
  • Pix é "somente por convite" — precisa pedir acesso e pode ser negado/demorar
  • Não suporta parcelamento de cartão no Brasil — limitação grave para e-commerce BR
  • Taxa de 3,99% + R$0,39 é a mais alta do mercado para cartão
  • Chargeback custa R$55 por contestação
Veredito: Excelente API mas o produto Brasil ainda não está pronto para operação completa com Pix + parcelamento.
