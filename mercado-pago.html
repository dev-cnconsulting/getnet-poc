Checkout Transparente via Orders API — Cartão de Crédito e Pix QR Code

⚡ D+0 Instantâneo 💸 Pix 0% 💳 Cartão 📦 SDK Oficial 🤖 MCP Server

Visão Geral

O Mercado Pago (Grupo Mercado Livre) oferece três modelos de checkout: Checkout Pro (hosted, redireciona), Checkout Bricks (componentes embed) e Checkout Transparente (API direta, controle total). Esta POC foca no Checkout Transparente via Orders API — a nova API unificada.

✅ Principais Vantagens
  • D+0 instantâneo — dinheiro cai na conta na hora
  • Pix 0% — sem taxa para receber via Pix
  • Onboarding em minutos — crie conta e comece a testar
  • SDK oficial em JS, PHP, Python, .NET, Java, Go
  • MCP Server para integração com IA
  • Discord community ativa

1. Credenciais & Ambientes

O Mercado Pago usa credenciais de teste e credenciais de produção, ambas obtidas no painel "Suas Integrações".

TipoDescrição
public_keyChave pública — usada no frontend para tokenizar cartão
access_tokenChave secreta — usada no backend para criar pagamentos
Base URLhttps://api.mercadopago.com (produção) / https://api.mercadopago.com (teste com credenciais de teste)

Como obter

  1. Acesse Mercado Pago Developers
  2. Crie uma aplicação: Suas Integrações → Criar aplicação
  3. Selecione Pagamentos Online → Checkout Transparente → API de Orders
  4. Copie as credenciais de teste (automaticamente geradas)
  5. Para produção: complete verificação de identidade e pegue credenciais de produção

2. Tokenização de Cartão (Frontend)

Antes de criar o pagamento, tokenize o cartão no frontend usando o SDK do Mercado Pago com a public_key:

// Instalar: npm install @mercadopago/sdk-react
import { Payment, initMercadoPago } from '@mercadopago/sdk-react';

initMercadoPago({ publicKey: 'TEST-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx' });

// Tokenizar cartão
const token = await Payment.createToken({
  cardNumber: '5031433215406201',
  cardholderName: 'JOAO SILVA',
  cardExpirationMonth: '12',
  cardExpirationYear: '2030',
  securityCode: '123',
  identificationType: 'CPF',
  identificationNumber: '12345678900'
});
// → token.id = "1234567890abcdef..."
💡 Por que tokenizar no frontend? O PAN (número do cartão) nunca chega ao nosso backend — reduz escopo PCI DSS para SAQ-A. O token é seguro e descartável.

4. Pagamento via Pix QR Code

O Mercado Pago suporta Pix via Checkout Transparente (API direta) ou Checkout Pro (hosted). Para API direta, use a API de Orders com payment_method.type: "pix":

POSThttps://api.mercadopago.com/v1/orders

Requisição

curl --request POST \
  --url https://api.mercadopago.com/v1/orders \
  --header 'Authorization: Bearer {access_token}' \
  --header 'Content-Type: application/json' \
  --header 'X-Idempotency-Key: {unique_uuid}' \
  --data '{
  "type": "online",
  "processing_mode": "automatic",
  "total_amount": "100.00",
  "description": "Pedido PIX-001",
  "transactions": {
    "payments": [
      {
        "amount": "100.00",
        "payment_method": {
          "type": "pix",
          "expiration_time": 1800
        }
      }
    ]
  },
  "payer": {
    "email": "joao@example.com",
    "first_name": "João",
    "last_name": "Silva",
    "identification": {
      "type": "CPF",
      "number": "12345678900"
    }
  }
}'

Resposta (201 Created)

{
  "id": "ORD-0987654321",
  "status": "action_required",
  "status_detail": "waiting_payment",
  "transactions": {
    "payments": [{
      "status": "pending",
      "status_detail": "waiting_payment",
      "payment_method": {
        "type": "pix",
        "qr_code_base64": "iVBORw0KGgoAAAANSUhEUg...",
        "qr_code": "00020126580014br.gov.bcb.pix...",
        "expiration_time": 1800
      }
    }]
  }
}
💡 O que fazer com a resposta qr_code = string "copia e cola" · qr_code_base64 = imagem QR Code em base64. Exiba ambos para o cliente. Pix é assíncrono — receba confirmação via webhook.

5. Webhooks — Notificações

Configure a URL de notificação em Suas Integrações → Notificações. O Mercado Pago envia HTTP POST para sua URL quando o status muda.

Configurar via API

curl --request POST \
  --url https://api.mercadopago.com/v1/notification_url \
  --header 'Authorization: Bearer {access_token}' \
  --data '{ "url": "https://seu-backend.com/api/webhook/mercado-pago" }'

Exemplo de Webhook Recebido

{
  "action": "payment.updated",
  "data": {
    "id": "PAY-1234567890"
  }
}
// → Fazer GET /v1/payments/{id} para obter detalhes completos

Eventos principais

EventoDescrição
payment.createdPagamento criado
payment.updatedStatus atualizado (approved, rejected, etc.)
merchant_order.updatedOrder atualizada

10. Aspecto Comercial

Para contratação: criar conta no Mercado Pago (PF ou PJ), criar aplicação no portal de developers, obter credenciais. Sem mensalidade, sem adesão. Recebimento D+0 instantâneo na conta Mercado Pago. Saque para banco externo tem taxa (~R$3,67 por saque). Para detalhes completos, veja Comparativo.
