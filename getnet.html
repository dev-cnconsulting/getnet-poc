🔴 Integração Getnet

Global API — Cartão de Crédito e Pix QR Code via OAuth 2.0

🔐 OAuth 2.0 💳 Cartão ⚡ Pix 🛡️ 3DS + Antifraude 🏦 Santander

Visão Geral

A Getnet (Grupo Santander) oferece a Global API — uma REST API unificada para pagamentos online e presenciais. No Brasil suporta cartão (crédito/débito), Pix QR Code, boleto, recorrência, split, cofre (tokenização), antifraude e 3D Secure — todos inclusos sem custo adicional.

2. Autenticação (OAuth 2.0)

POST/authentication/oauth2/access_token
curl --request POST \
  --url https://api-sbx.globalgetnet.com/authentication/oauth2/access_token \
  --header 'authorization: Basic {BASE64(client_id:client_secret)}' \
  --header 'content-type: application/x-www-form-urlencoded' \
  --data 'grant_type=client_credentials'
// Resposta
{
  "access_token": "eyJ0eXAi...",
  "token_type": "Bearer",
  "expires_in": 3599
}

4. Pagamento via Pix QR Code

POST/dpm/payments-gwproxy/v2/payments/qrcode/pix
⚠️ Pix é Assíncrono A API gera o QR Code e retorna status: "WAITING". A confirmação chega via webhook.

Requisição

curl --request POST \
  --url https://api-sbx.globalgetnet.com/dpm/payments-gwproxy/v2/payments/qrcode/pix \
  --header 'authorization: Bearer {access_token}' \
  --header 'content-type: application/json' \
  --header 'x-qrcode-expiration-time: 180' \
  --data '{
  "amount": 10050,
  "currency": "BRL",
  "customer_id": "customer_21081826",
  "order_id": "Order-00123",
  "idempotency_key": "1eb2412c-165a-41cd-b1d9-76c575d30a21"
}'

Resposta (201 Created)

{
  "payment_id": "03ec0ede-3bc9-42dd-a71b-1c3a670b2b89",
  "status": "WAITING",
  "description": "QR Code successfully generated and awaiting payment.",
  "additional_data": {
    "qr_code": "00020101021226740014br.bcb.pix...",
    "creation_date_qrcode": "2025-10-20T18:55:55",
    "expiration_date_qrcode": "2025-10-20T18:58:58",
    "psp_code": "033"
  }
}

5. Webhooks — Notificações

Getnet usa dois modelos: Callbacks HTTP GET (4 URLs configuradas na acreditação) e Webhooks HTTP POST (configuráveis via API).

Parâmetros PIX (Callback GET)

ParâmetroDescrição
payment_typepix
payment_idUUID da transação
statusAPPROVED DENIED ERROR
transaction_idID no PSP
payer_nameNome do pagador (se aprovado)
payer_cpfCPF do pagador
⚠️ Retry Policy Em caso de erro, Getnet reenvia a cada 15 minutos, até 4 tentativas.
