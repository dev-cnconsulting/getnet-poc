Visão Geral

O Pagar.me (Grupo Stone) é o PSP (Payment Service Provider) mais robusto do Brasil para desenvolvedores. API REST seguindo os melhores padrões, SDKs em 5+ linguagens, checkout hosted ou transparente, split de pagamento nativo e retentativa automática de vendas recusadas.

✅ Principais Vantagens
  • API REST de primeira — docs.pagar.me excelentes com llms.txt para IA
  • Split de pagamento nativo — marketplace-ready, split com N recebedores
  • Retentativa transparente — salva 5-10% de vendas recusadas automaticamente
  • Pix completo (QR Code + estorno + split via Pix)
  • Checkout hosted (redirect) ou transparente (API direta)
  • SDKs em JS, PHP, Python, Ruby, Go
  • Cobrança pontual (order) e recorrente (subscription)

1. Credenciais & Autenticação

O Pagar.me usa HTTP Basic Auth com a secret_key como único campo (password vazio):

// Header obrigatório em TODAS as requisições
Authorization: Basic {BASE64("sk_test_xxx:")}
User-Agent: pagarme-skill-generated/1.0

// Exemplo curl
curl --request POST https://sdx-api.pagar.me/core/v5/orders \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Basic c2tfdGVzdF9hYmN4ZGVmOg==' \
  --header 'User-Agent: pagarme-skill-generated/1.0'

2. Pagamento por Cartão de Crédito

💡 Tokenização via Checkout Para não lidar com dados de cartão no backend, use o Checkout Pagar.me via POST /paymentlinks — cria uma URL de checkout hosted. O cliente paga na página do Pagar.me e o webhook confirma.

3. Pagamento via Pix QR Code

O Pagar.me suporta Pix como meio de pagamento em pedidos. Crie um order com payment_method: "pix" e receba o QR Code:

POST/orders

Requisição

curl --request POST \
  --url https://sdx-api.pagar.me/core/v5/orders \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Basic {base64(sk_test:)}' \
  --data '{
  "code": "ORDER-PIX-001",
  "amount": 10050,
  "currency": "BRL",
  "customer": {
    "name": "João Silva",
    "email": "joao@example.com",
    "document": "12345678900",
    "document_type": "CPF",
    "type": "individual"
  },
  "payments": [
    {
      "payment_method": "pix",
      "pix": {
        "expires_in": 1800
      }
    }
  ]
}'

Resposta (201 Created)

{
  "id": "or_YYYYYYYYYYYY",
  "code": "ORDER-PIX-001",
  "status": "pending",
  "charges": [{
    "id": "ch_YYYYYYYY",
    "status": "pending",
    "last_transaction": {
      "status": "with_qr_code",
      "qr_code": "00020126580014br.gov.bcb.pix...",
      "qr_code_url": "data:image/png;base64,...",
      "expires_at": "2025-10-20T19:25:55Z"
    }
  }]
}
💡 Pix via Pagar.me qr_code = string "copia e cola" · qr_code_url = imagem base64 do QR Code. Pix é assíncrono — receba confirmação via webhook. Pix no Pagar.me também suporta estorno (até 90 dias) e split.

4. Checkout Hosted (Link de Pagamento)

Para evitar lidar com dados de cartão, crie um link de checkout:

POST/paymentlinks
curl --request POST \
  --url https://sdx-api.pagar.me/core/v5/paymentlinks \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Basic {base64(sk_test:)}' \
  --data '{
  "type": "order",
  "amount": 10000,
  "currency": "BRL",
  "enabled_payment_methods": ["credit_card", "pix", "boleto"]
}'
// → { "url": "https://checkout.pagar.me/..." }

5. Split de Pagamento

O Pagar.me tem o melhor split do mercado — nativo, suporta N recebedores com regras complexas:

{
  "payments": [{
    "payment_method": "credit_card",
    "credit_card": { /* dados do cartão */ },
    "split": [
      { "recipient_id": "re_xxx1", "amount": 7000, "type": "flat" },
      { "recipient_id": "re_xxx2", "amount": 3000, "type": "flat" }
    ]
  }]
}

6. Webhooks — Notificações

Configure a URL de webhook no painel do Pagar.me ou via API. O Pagar.me envia HTTP POST com o evento:

Configurar via API

curl --request POST \
  --url https://sdx-api.pagar.me/core/v5/webhooks \
  --header 'Authorization: Basic {base64(sk_test:)}' \
  --data '{ "url": "https://seu-backend.com/api/webhook/pagarme" }'

Exemplo de Webhook Recebido

{
  "id": "evt_xxxxxxxx",
  "type": "order.paid",
  "data": {
    "id": "or_xxxxxxxx",
    "code": "ORDER-001",
    "status": "paid",
    "amount": 10000
  }
}
